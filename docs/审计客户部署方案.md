# SAP BASIS 审计自动化工具 — 客户系统部署方案

## 一、核心策略：申请专用审计账号

### 1.1 为什么申请专用账号而非直接部署

客户对敏感生产系统中运行自动化工具天然存在抵触，直接部署面临以下阻力：
- 安全团队对"自动化脚本"的合规性质疑
- BASIS 团队担心工具误操作影响生产
- 审计方与被审计方的信任博弈

**最佳路径：向客户申请一个专用只读账号**，该账号覆盖 AC（应用控制）、GC（通用控制）和 BASIS 测试所需的全部**查看权限**，以"人工也可执行、工具仅替代重复劳动"的逻辑说服客户。

### 1.2 权限清单

#### 1.2.1 事务代码（T-codes）

| T-code | 用途 | 对应审计检查项 |
|--------|------|---------------|
| **SE16 / SE16N** | 数据浏览器 — 查看表数据 | T000(客户端)、TDDAT、USR02、USR04、USR41、AGR_USERS、AGR_1251 等 |
| **SA38** | 程序执行 — 运行报表程序 | 执行 `RSUSR002`、`RSUSR200` 等标准用户审计报表 |
| **PFCG** | 角色维护（只读） | 查看角色菜单、权限参数文件结构 |
| **SUIM** | 用户信息系统 | 用户/角色/参数文件/权限值/更改文档的复合查询入口 |
| **S_BCE_68001397** | SUIM → 用户按复杂条件选择 → 按权限值 | 权限值级用户分配情况 |
| **S_BCE_68001400** | SUIM → 用户按复杂条件选择 → 按角色/参数文件 | 角色级用户分配情况 |
| **S_BCE_68001439** | SUIM → 更改文档 → 用户 → 按角色/参数文件 | 用户分配变更历史 |
| **S_BCE_68001440** | SUIM → 更改文档 → 针对参数文件 | 参数文件变更历史 |
| **S_ALR_87012277** | SAP 标准报表 | 用户主记录基础信息 |
| **RSSCD100_PFCG** | 角色更改文档 | 角色修改历史追溯 |

#### 1.2.2 审计补充查看权限（需额外申请）

| T-code | 用途 |
|--------|------|
| **SM20 / SM20N** | 安全审计日志 — 查看系统安全事件（登录失败、权限违规等） |
| **SCU3** | 表变更日志 — 查看自定义表的数据修改记录 |
| **SE01 / SE09 / SE10** | 传输组织器 — 查看程序变更和传输请求历史 |
| **SE11** | ABAP 数据字典 — 查看表结构定义 |
| **SE38** | ABAP 编辑器（只读） — 查看程序源代码 |
| **SE37** | 函数构建器（只读） — 查看函数模块代码 |
| **SE80** | 对象导航器（只读） — 浏览程序、功能组、类等开发对象 |
| **ST03 / ST03N** | 工作负载分析 — 查看系统使用统计 |
| **SM50 / SM66** | 进程概览 — 查看当前活动工作进程 |
| **SM37** | 作业概览 — 查看后台作业执行情况 |
| **SM12** | 锁条目 — 查看表锁定情况 |
| **SM21** | 系统日志 — 查看系统级错误和警告 |

#### 1.2.3 权限对象（Authorization Objects）

| 权限对象 | 值 | 用途 |
|----------|-----|------|
| **S_TCODE** | 上述所有 T-code | 允许执行指定事务代码 |
| **S_TABU_DIS** | 所需表名列表 | 允许 SE16 查看指定表（含 ACTVT=03 显示） |
| **S_TABU_CLI** | X | 跨客户端表访问（若需查看跨客户端自定义表） |
| **S_TABU_NAM** | 所需表名 | 按表名授权访问 |
| **S_DEVELOP** | ACTVT=08（显示） | 允许查看 ABAP 程序源码 |
| **S_ADMI_FCD** | 受限管理功能 | 查看系统日志等功能 |

#### 1.2.4 账户类型建议

```
账户类型：System 类型（非 Dialog 类型，避免占用对话进程）
用户组：  AUDIT（新建审计专用用户组）
有效期：  审计期间 + 1个月缓冲
锁定策略：非使用时段自动锁定
```

## 二、沟通策略：消除客户顾虑

### 2.1 工具原理透明化

向客户出具正式说明文档，核心要点：

> **本工具不是外接系统、不修改任何数据、不建立持久连接。**
>
> 其工作原理等同于一名审计人员坐在 SAP GUI 前，按审计程序清单逐项操作并截屏保存。区别仅在于 —— 由 Python 脚本驱动 pyautogui 库模拟键盘输入和截屏操作，将原本 8 小时的人工重复操作压缩至 2 小时。

**技术要点说明：**

| 客户可能的顾虑 | 实际情况 |
|---------------|---------|
| 是否接入 SAP 后台数据库？ | 否。仅通过 SAP GUI 前端界面操作，与人工操作完全一致 |
| 是否会修改配置/数据？ | 否。所有 T-code 均为查询/显示类，且账号仅有只读权限 |
| 是否有网络安全隐患？ | 脚本在隔离的备用机上本地运行，不连外网 |
| 是否会影响系统性能？ | 单会话、单线程执行，负载等同于一名在线用户 |
| 是否涉及数据导出？ | 仅截屏保存为 PNG 图片，不导出可被二次加工的数据文件 |

### 2.2 视频演示

建议录制 3-5 分钟的操作演示视频，展示：
1. 工具启动 → 输入审计期间参数 → 选项勾选
2. 实际运行过程中的自动截屏画面（如 SE16 查 T000、SUIM 权限查询等）
3. 产出物展示：按步骤归类的 PNG 截图文件夹结构
4. 断点续连功能：模拟中途关闭后重新执行，从断点继续

### 2.3 签署文件建议

- **保密协议补充条款**：明确工具用途、数据不离开客户环境
- **操作时间窗口确认**：约定仅在非业务高峰期运行（如 18:00-次日 08:00 或周末）

## 三、测试环境先行策略

### 3.1 推进路径

```
测试环境（DEV/QAS）验证 → 客户 BASIS 团队确认无影响 → 生产环境（PRD）正式执行
```

### 3.2 测试环境注意事项

测试环境同样需要考虑数据敏感性：
- **QAS 系统**：通常为生产数据副本，按生产标准处理
- **DEV 系统**：可能使用脱敏数据，可作为首站验证
- 优先争取在 **QAS 系统** 执行，因为数据结构与生产一致、截图结果具有审计参考价值

### 3.3 测试环境执行清单

| 阶段 | 内容 | 产出 |
|------|------|------|
| Day 1 | 部署工具、申请账号、安装 Python 依赖 | 环境就绪确认单 |
| Day 2 | 执行完整流程（16 个步骤），观察有无报错 | 测试运行日志 |
| Day 3 | BASIS 团队复核截屏内容准确性 | 确认函 |
| Day 4 | 提交测试环境运行报告，申请生产执行 | 测试通过报告 |

## 四、运行环境要求

### 4.1 硬件要求

| 项目 | 要求 | 说明 |
|------|------|------|
| **备用机** | Windows 10/11 专业版 | 建议客户提供专用笔记本电脑，审计结束后格式化归还 |
| 内存 | ≥ 8GB | |
| 硬盘 | ≥ 20GB 可用空间 | 截屏图片存储（预计 500-2000 张 PNG） |
| 屏幕分辨率 | ≥ 1920×1080 | 确保 SAP GUI 元素不被遮挡 |
| 网络 | 内网连接 SAP 服务器 | **无需外网** |

### 4.2 软件要求

| 软件 | 版本 | 说明 |
|------|------|------|
| SAP GUI for Windows | 7.70+ 或 8.00 | 预装并配置好连接 |
| Python | 3.9+ | 安装包：`pyautogui`, `pillow`, `tkinter` |
| 截图保存目录 | 本地路径 | 建议 `D:\audit_output\YYYYMMDD\` |

### 4.3 运行时间预估

| 步骤范围 | 内容 | 预估耗时 |
|----------|------|---------|
| Getbasis_1 ~ Getbasis_4 | 客户端信息、用户锁定、密码参数、登录限制 | ~15 分钟 |
| Getbasis_5 ~ Getbasis_8 | 用户默认值、Internet用户、自定义字段、更改文档 | ~20 分钟 |
| Getbasis_9 | 高权限角色分析（9个子步骤，含PFCG查看） | ~40 分钟 |
| Getbasis_10 ~ Getbasis_13 | 用户比对、关键权限、接口用户、系统参数 | ~25 分钟 |
| Getbasis_14 ~ Getbasis_16 | USR02报告、SAP_ALL/SAP_NEW、汇总 | ~20 分钟 |
| **合计** | | **约 2 小时** |

> ⚠️ **注意**：以上不含 JETB（日志分析工具）的日志导出时间。若需额外执行 SM20 日志导出，需另加 30-60 分钟，取决于日志量。

### 4.4 SAP GUI 配置要求

```
SAP GUI 主题：   使用默认 SAP 主题（非自定义皮肤）
字体缩放：       100%（不缩放）
SAP 快捷方式：   提前配置好目标系统的快捷方式
自动关闭弹窗：   关闭 SAP GUI 的系统消息弹窗
屏幕保护/锁屏：   禁用（运行期间不能锁屏）
电源管理：       设置为"从不休眠"
```

## 六、风险应对预案

| 风险 | 概率 | 应对措施 |
|------|------|---------|
| 客户拒绝提供账号 | 中 | 降级为人工操作方案，但强调工具可提升一致性和完整性 |
| SAP GUI 版本不兼容 | 低 | 提前确认客户 SAP GUI 版本，适配不同版本的按钮位置 |
| 运行中途网络断开 | 中 | **断点续连机制**：重连后自动从断点继续，无需重新开始 |
| 大表查询超时 | 中 | 已内置 `SLEEP_LONG=20s` 超长等待配置，可按需调大 |
| 截图遗漏 | 低 | 断点续连 + 子步骤标记机制保证每个检查项都被覆盖 |
| 客户担心数据外泄 | 高 | 备用机不连外网、截图仅 PNG 格式、审计结束即清除 |

## 七、FAQ（客户常见问题预答复）

**Q1: 为什么不能直接给我们 SQL 脚本去数据库查？**
A: 部分审计证据必须来自 SAP 应用层界面（如 SUIM 的权限查询结果、PFCG 的角色层级结构），数据库表层面无法展示完整的业务逻辑和权限继承关系。另外，截屏保存 SAP GUI 原生界面也是审计证据链的重要环节。

**Q2: 工具出错了会不会误删数据？**
A: 工具的 T-code 全部为查询/显示类，账号也只有只读权限，即使脚本出错（如输入了预期外的字符），在权限层面也无法执行任何写操作。最坏情况是截了一张空白画面 —— 断点续连会捕获并重试。

**Q3: 能不能在生产系统业务时间运行？**
A: 理论上可以（单用户负载极低），但建议在非高峰期运行，一方面避免 BASIS 团队担忧，另一方面确保 SAP GUI 响应速度稳定、截图不会因网络波动而失败。

**Q4: 截图文件怎么交接？**
A: 运行结束后，所有截图按步骤分文件夹存放于本地。通过客户指定的安全渠道（如内网共享目录/USB加密盘）交接，备用机上不留副本。
